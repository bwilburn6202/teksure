## Tip of the Day: Use SSH Config Files to Simplify Remote Connections

**Category:** Productivity / Developer Tools

---

If you regularly SSH into multiple servers, typing out long hostnames, usernames, and port numbers gets tedious fast. Your **SSH config file** (`~/.ssh/config`) lets you define aliases for each server so you can connect with a single short command.

### How to set it up:

Open (or create) `~/.ssh/config` and add an entry like this:

```
Host myserver
    HostName 192.168.1.100
    User john
    Port 2222
    IdentityFile ~/.ssh/id_rsa
```

Now instead of typing:

```bash
ssh john@192.168.1.100 -p 2222 -i ~/.ssh/id_rsa
```

You just type:

```bash
ssh myserver
```

### Bonus Tips:
- Use `Host *` at the top of the file to set global defaults (e.g., `ServerAliveInterval 60` to prevent timeout disconnections).
- You can add as many `Host` blocks as you need.
- Works with `scp`, `rsync`, and many other tools that use SSH under the hood.
